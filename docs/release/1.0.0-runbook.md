# Binderbase v1.0.0 — Release Runbook

> **Audience:** Matt (repository owner). This runbook was prepared by the
> Copilot agent after Phases 1–3 merged to `main`.

---

## Pre-tag gate

- [ ] Phase 1–3 PRs merged to `main`.
- [ ] `main` CI green:
  - `CI / Frontend`
  - `CI / Rust lint`
  - `CI / Rust test (ubuntu-latest)`
  - `CI / Rust test (macos-latest)`
  - `CI / Rust test (windows-latest)`
  - `CI / Security audit`
  - `CodeQL / Analyze`
  - `cargo-deny / cargo-deny`
  - `OSV-Scanner / OSV vulnerability scan`
- [ ] Local smoke test: `npm run tauri dev` launches, Import / Export tab
  renders, Catalog Import buttons are visible.
- [ ] `git status` clean; `git pull --ff-only origin main`.

---

## Tag

```bash
git tag -s v1.0.0 -m "Binderbase 1.0.0"
git push origin v1.0.0
```

> `-s` signs the tag with your GPG key if configured. Fall back to `-a`
> (annotated, unsigned) if GPG is not set up. The tag push triggers the
> release workflow regardless of tag signing.

---

## Watch

- [ ] Actions → **Release** workflow runs all three jobs:
  - `build` (matrix: ubuntu-latest, macos-latest, windows-latest)
  - `checksums`
  - `sign`
- [ ] All three OS builds succeed.
- [ ] `SHA256SUMS.txt` and every `.bundle` file upload to the draft release.

---

## Publish

- [ ] Open the draft release on the
  [Releases page](https://github.com/aecyx/Binderbase/releases).
- [ ] Paste the body from `.github/release-template.md`, filling in the
  `<!-- Brief summary -->` placeholder with the CHANGELOG `[1.0.0]` content:

  > **Stable release.** Promotes 1.0.0-rc.1 to stable.
  >
  > - E2E Playwright coverage added for the catalog import flow:
  >   start → progress → cancel → restart.
  > - End-to-end cosign verification of release artifacts documented.
  > - Branch protection on `main` tightened with Copilot code review.

- [ ] Untick **"Set as a pre-release"** (the workflow defaults this to `false`
  for non-rc tags — verify that the checkbox is already unchecked).
- [ ] Click **"Publish release."**

## Post-publish verification

- [ ] From a clean directory, re-run the full cosign verification transcript
  from `docs/release/cosign-verification.md` against the **v1.0.0** artifacts
  (replace `1.0.0-rc.1` with `1.0.0` in all file names):

  ```bash
  mkdir verify-v1.0.0 && cd verify-v1.0.0
  gh release download v1.0.0

  # Checksums
  sha256sum -c SHA256SUMS.txt

  # Cosign (repeat for each artifact)
  cosign verify-blob \
    --bundle <artifact>.bundle \
    --certificate-oidc-issuer https://token.actions.githubusercontent.com \
    --certificate-identity-regexp "^https://github\\.com/aecyx/Binderbase/\\.github/workflows/release\\.yml@refs/tags/.*$" \
    <artifact>
  ```

- [ ] If any step fails, the release is broken. Delete the GitHub release,
  retag via force-delete-and-repush, fix the root cause, and start over:

  ```bash
  git tag -d v1.0.0
  git push origin :refs/tags/v1.0.0
  ```

  Document the retag in CHANGELOG `[Unreleased]`.

- [ ] Wait ~24 hours, then re-run Scorecard via `workflow_dispatch` on
  `scorecard.yml`. Record the new score. Expected: `Signed-Releases` moves
  from 0 → ≥ 7 (the check wants five most recent releases signed; v1.0.0 +
  v1.0.0-rc.1 = two signed → partial credit).
