## Verifying downloads

Each published binary artifact (`.msi`, `.dmg`, `.deb`, `.AppImage`) has:

1. A **SHA-256 checksum** entry in `SHA256SUMS.txt`
2. A **cosign signature** (Sigstore keyless, accompanying `.bundle` file)

### 1. Verify SHA-256 checksum

Download `SHA256SUMS.txt` from the release page, then:

**Linux:**

```bash
sha256sum -c SHA256SUMS.txt
```

**macOS:**

```bash
shasum -a 256 -c SHA256SUMS.txt
```

**Windows (PowerShell):**

```powershell
# Replace <downloaded-file-name> with the actual file you downloaded:
(Get-FileHash .\<downloaded-file-name> -Algorithm SHA256).Hash
# Compare against the hash in SHA256SUMS.txt
```

### 2. Verify cosign signature

Install [cosign](https://docs.sigstore.dev/cosign/system_config/installation/)
(v3+), then verify any artifact against its `.bundle`:

```bash
cosign verify-blob \
  --bundle <artifact>.bundle \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  --certificate-identity-regexp "^https://github\\.com/aecyx/Binderbase/\\.github/workflows/release\\.yml@refs/tags/.*$" \
  <artifact>
```

A successful verification prints `Verified OK`. This proves the artifact was
built by the Binderbase GitHub Actions release workflow and has not been
tampered with since signing.

## Known issues
